# IASNE 产品与技术重构文档

## 文档目标

本套文档用于支持 `IASNE`（前端）与 `backend`（后端）在不丢失现有功能的前提下，进行一次以**性能提升**和**透明度提升**为核心的重构。

重构范围覆盖：
- 前端：`IASNE/src/**`
- 后端：`backend/app/**`
- 接口契约：`IASNE/src/fetch/apis.js` 与 `backend/app/modules/*/router.py`

## 核心结论

1. 当前系统已具备完整业务闭环：用户/公司/船舶、上传、统计、优化、状态提醒、航线优化、CII计算。
2. 性能瓶颈主要集中在：前端请求与渲染策略、后端重型算法和数据库访问路径。
3. 透明度短板主要集中在：缺少统一可观测性标准、API契约可追踪性不足、重构过程验收口径不一致。
4. 推荐采用“兼容优先 + 渐进替换”策略，通过 BFF/Adapter、双写与灰度逐步完成升级。

## 代码盘点快照

- 前端视图组件：约 24 个（`IASNE/src/views`）
- 后端模块路由：10 个（`backend/app/modules/*/router.py`）
- 后端接口定义：约 46 个 `@api.*` 端点
