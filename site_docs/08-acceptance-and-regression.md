# 验收与回归

## 一、验收框架

验收采用“三层验证”模型：

1. 契约层
- API 请求/响应兼容
- 错误码语义兼容

2. 业务层
- 功能流程完整
- 指标口径一致

3. 性能与稳定性层
- 延迟、吞吐、错误率达标
- 异常场景可恢复

## 二、回归测试清单

### 核心流程

1. 注册 -> 登录 -> 选择公司/船舶
2. 上传 CSV -> 数据处理 -> 历史可查
3. 统计页面查询（频次/关系/时序）
4. 优化建议（航速、吃水差）
5. 状态提醒（船体/发动机）
6. 航线规划（最短、油耗最优、历史对比）
7. CII 计算器

### 接口兼容

1. `/api/*` 入口可用
2. 旧路径兼容窗口可用（含 deprecation 标识）
3. 新路径全量可用

### 数据一致性

1. 同样输入下，新旧输出差异在容差内
2. 关键统计指标与历史版本一致

## 三、性能验收建议

1. 指标
- API P50/P95/P99
- 错误率
- 缓存命中率
- 队列任务成功率

2. 场景
- 统计高并发读取
- 上传高峰
- 航线优化并发任务

3. 发布门禁
- 不满足阈值禁止发布

## 四、回滚策略

1. 功能开关回滚
2. 路由级回滚（新旧服务切换）
3. 数据回滚（只读保护 + 双写校验）
4. 文档化演练（每次大版本前执行）
