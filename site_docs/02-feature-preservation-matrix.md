# 功能保留矩阵

## 保留原则

1. 用户可感知功能不减少
2. API 对外契约在重构期保持兼容
3. 核心计算结果（允许小误差阈值）保持一致
4. 历史数据可追溯

## 模块级保留清单

| 模块 | 当前能力 | 保留要求 | 验收方式 |
|---|---|---|---|
| 用户管理 | 注册、登录、用户CRUD | 请求参数与返回结构兼容 | API 合约测试 + UI 冒烟 |
| 公司管理 | 公司CRUD、公司船舶查询 | 同步保留分页/筛选行为 | API 快照对比 |
| 船舶管理 | 船舶CRUD、详情、列表附带CII | 字段语义与排序策略不变 | 对比测试 |
| 数据上传 | CSV上传、解析、入库、衍生计算 | 上传流程稳定，失败可追踪 | 大文件压测 + 回放 |
| 数据统计 | 属性分布/关系/时序、油耗、完整度 | 统计口径不变 | SQL 对账 + 图表对比 |
| 优化建议 | 航速优化、吃水差优化 | 优化表格与关键指标不退化 | 结果容差测试 |
| 状态提醒 | 船体/螺旋桨、发动机状态 | 关键指标定义不变 | 指标核对 |
| 航线优化 | 最短路径、油耗优化、历史航迹 | 路径可解释、耗时可观测 | 回归用例 + 性能基线 |
| CII计算器 | 独立 CII 计算 | 输入输出兼容 | 黄金样本测试 |

## API 兼容重点

1. 保留 `/api/*` 访问习惯（由网关/BFF统一处理）
2. 保留历史 typo 路径兼容窗口（如 `optimazation`），并输出 deprecation 日志
3. 所有旧接口给出迁移目标与下线时间表

## 数据口径兼容重点

1. CII、油耗、速度、吃水差等关键指标公式统一在“指标字典”中落盘
2. 每次变更必须附带“旧版 vs 新版”差异报告
3. 对涉及 ML/优化模型的结果，定义容差区间与异常报警阈值
